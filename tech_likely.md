## Cheap backups

date: 2023-11-20, from: Alex Schroeder's Blog

<h1>Cheap backups</h1>

<p>Every day is backup day. I use BorgBackup to backup up my laptop to one of two external disks. One of these is always at the office, so there are three copies: on my laptop, on a backup disk at home and on a backup disk at the office. It&rsquo;s important that those three are never in the same location.</p>

<p>I do concede, however, that it&rsquo;s tricky setup.</p>

<p>For my wife, I&rsquo;ve set up the Mac to backup the laptop and an external disk with media files to one of two external disks. Again, one of these is always at the office. So, same deal, except now I&rsquo;m using TimeMachine instead of BorgBackup. Somehow, that makes it take a very long time. But it seems to work.</p>

<p>Still, plugging in backup disks, carrying them to the office, bringing the other one back, plugging it in again‚Ä¶ as you can imagine, we don&rsquo;t do this nearly often enough.</p>

<p>So I need a quick way to backup stuff. Like, super quick. Super cheap.</p>

<p>The simplest option I can think of is to use rsync on the local disk because it has an option to create links for files ¬≠‚Äì and while a link doesn&rsquo;t take zero space, it takes a lot less space than making a copy.</p>

<p>So what I wanted was a quick command that I can run in the directory I&rsquo;m in and it creates a backup using rsync.</p>

<p>I&rsquo;ve used the following bash script. Let me know if it can be improved.</p>

<pre><code>#!/usr/bin/bash
d=$(basename $(pwd))
t=$(date --iso-8601)
echo Creating a snapshot of $d in ../$d-$t
rsync --link-dest &quot;../$d&quot; --archive . &quot;../$d-$t/&quot;
</code></pre>

<p>That is, if the current directory is <code>/home/alex/src/wiki</code>:</p>

<ul>
<li>it creates the backup in <code>/home/alex/src/wiki-2023-11-19</code> (appending today&rsquo;s date)</li>
<li>from there, the links all point to the original (where the command was invoked)</li>
</ul>

<p>When I use <code>ls -l</code>, this is what it looks like:</p>

<pre><code>drwxr-xr-x 67 alex alex 749568 18. Nov 23:56 wiki/
drwxr-xr-x 67 alex alex 548864 17. Okt 23:43 wiki-2023-10-17/
drwxr-xr-x 67 alex alex 548864 19. Okt 22:47 wiki-2023-10-19/
drwxr-xr-x 67 alex alex 552960 30. Okt 13:18 wiki-2023-10-30/
</code></pre>

<p>It&rsquo;s not great, but it works.</p>

<p>For a file that hasn&rsquo;t changed, you&rsquo;ll see that <code>ls -l</code> says there are four links to this file. This is correct: one link for every directory. The data exists only once, on the disk.</p>

<pre><code>&gt; ls -l w*/2000-03-10_Elendor.md 
-rw-r--r-- 4 alex alex 16313 28. Apr 2017  wiki/2000-03-10_Elendor.md
-rw-r--r-- 4 alex alex 16313 28. Apr 2017  wiki-2023-10-17/2000-03-10_Elendor.md
-rw-r--r-- 4 alex alex 16313 28. Apr 2017  wiki-2023-10-19/2000-03-10_Elendor.md
-rw-r--r-- 4 alex alex 16313 28. Apr 2017  wiki-2023-10-30/2000-03-10_Elendor.md
</code></pre>

<p>For a file that changes all the time, <code>ls -l</code> shows that there is just one link for every file, meaning that these files are distinct. And for a lay person, that is confirmed by the different file size and last modified timestamp.</p>

<pre><code>&gt; ls -l w*/changes.md
-rw-r--r-- 1 alex alex 1997 17. Okt 16:28 wiki-2023-10-17/changes.md
-rw-r--r-- 1 alex alex 2123 19. Okt 22:36 wiki-2023-10-19/changes.md
-rw-r--r-- 1 alex alex 5847 30. Okt 13:18 wiki-2023-10-30/changes.md
-rw-r--r-- 1 alex alex 4255 18. Nov 23:54 wiki/changes.md
</code></pre>

<p><a class="tag" href="/search/?q=%23Administration">#Administration</a> <a class="tag" href="/search/?q=%23Backup">#Backup</a></p>

<h2>Limitations</h2>

<p>As <a class="account" href="https://functional.cafe/@edavies" title="@edavies@functional.cafe">@edavies</a> and <a class="account" href="https://idiomdrottning.org/users/Sandra" title="@Sandra@idiomdrottning.org">@Sandra</a> have pointed out, this only works if the editing you do always recreates the files you&rsquo;re editing (because then these files get a new inode even if their names stay the same). Examples where this is a problem: database files are updated in place; some editors allow you to decide how backups are created. Emacs, for example, has the option <code>backup-by-copying</code> which is nil by default. When you save a file, the original is renamed to the backup and a new file is written. If you set the option, however, the backup is copied first, and then the original is changed in-place (like a database file, without getting a new inode). Now you&rsquo;ve changed the file in all the &ldquo;cheap backup&rdquo; directories, since the links keep pointing to the same inode and that inode now points to the new file.</p>

<p>Possibly a better solution that uses copies for the first backup and then future backups link to the old backups (and never to the originals) would be the following:</p>

<pre><code>#!/usr/bin/bash
d=$(basename $(pwd))
t=$(date --iso-8601)
p=$(find .. -maxdepth 1 -type d -name &quot;$d-[0-9][0-9][0-9][0-9]-[0-9][0-9]-[0-9][0-9]&quot; -prune | sort | tail -n 1)
if [ -z &quot;$p&quot; ]; then
  echo Creating a snapshot of $d in ../$d-$t
  rsync --archive . &quot;../$d-$t/&quot;
else
  echo Creating a snapshot of $d in ../$d-$t with links into $p
  rsync --link-dest &quot;$p&quot; --archive . &quot;../$d-$t/&quot;
fi
</code></pre>

<p>Note that now, in case you just need a single backup, you might just as well just make a copy using your favourite file manager. Not so cheap any more, eh? ü§®</p>

<p>Or use <a href="https://github.com/laurent22/rsync-time-backup/tree/master">rsync-time-backup</a>.</p>

<h2>Another solution</h2>

<p><a class="account" href="https://functional.cafe/@edavies" title="@edavies@functional.cafe">@edavies</a> offered the following script for a use case for <code>/home</code>, <code>/etc</code> and <code>/usr/local</code> to a 1TB hard drive by the computer:</p>

<pre><code>#!/bin/bash

set -e

STAMP=$(date &quot;+%Y-%m-%dT%H:%M:%S&quot;)
DEST=/run/media/ed/hitachi
MACHINE=$(uname -n)
TARGET=$DEST/$MACHINE

[ -d $TARGET ] || (
    mkdir $TARGET
    mkdir $TARGET/0
    ln -s ../0 $TARGET/current
)

[ -L $TARGET/current ] || (
    echo Target directory $TARGET/current doesn\'t exist or isn\'t a symlink
    false
)

for DIR in home etc usr/local
do
    mkdir -p $TARGET/incomplete-$STAMP/$DIR
    rsync -axv --link-dest=$TARGET/current/$DIR/ /$DIR/ $TARGET/incomplete-$STAMP/$DIR/
done

mv $TARGET/incomplete-$STAMP $TARGET/back-$STAMP
rm -f $TARGET/current
ln -s back-$STAMP $TARGET/current
ls -l $TARGET
df -h $DEST
umount $DEST
</code></pre>

## OpenBSD formal driver verification with SeL4

date: 2023-11-20, from: OS News

The seL4 microkernel is currently the only kernel that has been fully formally verified. In general, the increased interest in ensuring the security of a kernel&#8217;s code results from its important role in the entire operating system. One of the basic features of an operating system is that it abstracts the handling of devices. This abstraction is represented by device drivers &#8211; the software that manages the hardware. A proper verification of the software component could ensure that the device would work properly unless there is a hardware failure. In this paper, we choose to model the behavior of a device driver and build the proof that the code implementation matches the expected behavior. The proof was written in Isabelle/HOL, the code translation from C to Isabelle was done automatically by the use of the C-to-Isabelle Parser and AutoCorres tools. We choose Isabelle theorem prover because its efficiency was already shown through the verification of seL4 microkernel. ## On Ad Blockers

date: 2023-11-19, from: Manu - I write blog

<p>Every time I stumble on a discussion about blocking ads on the web I ask myself if there even is a compelling argument against it. I block ads on the web. Safari is set to prevent cross-site tracking and to hide my IP, I have <a href="https://1blocker.com" rel="noreferrer" target="_blank">1Blocker</a> running on both my Mac and my iPhone and I also have <a href="https://nextdns.io" rel="noreferrer" target="_blank">NextDNS</a> enabled. If I can prevent ads and tracking from showing up on my device, I'm gonna do it. Why? Because ads provide literally no value to my life.</p>
<p>But that's the easy part. Things become a lot more complex when you start dealing with all the other factors that are attached to the ads world. For example, you can find this kind of argument:</p>
<blockquote>
<p>People who don't pay and block ads are scamming YouTube, or scamming advertisers.</p>
</blockquote>
<p>Or even this kind of argument:</p>
<blockquote>
<p>And more importantly, they are scamming the content creators. They are not usually a massive company, but working alone, with YouTube revenue as their main income.</p>
</blockquote>
<p>Am I scamming YouTube because I block their awful ads? Would I be scamming YouTube if I muted the video and looked at something else while ads were playing? What's the scam here? Who is getting scammed?</p>
<p>One might even argue that I'm doing YouTube a favor and saving them bandwidth by not allowing pointless ads to be served to me. Also, scamming the content creators? If I pay for YouTube Premium I'd still be served in video ads by the creators themselves. Should I be complaining? Am I getting scammed then? I'm paying for no ads after all.</p>
<p>No matter the case, ads on the web will always be tricky and live in this bizarre gray area. And until someone comes up with a really compelling argument for allowing ads I'll keep running ad-blockers because I just can't stand ads.</p>

## Campaign building blocks

date: 2023-11-19, from: Alex Schroeder's Blog

<h1>Campaign building blocks</h1>

<p><a class="account" href="https://ttrpg-hangout.social/@enfors" title="@enfors@ttrpg-hangout.social">@enfors</a> recently wondered about &ldquo;important aspects of sandboxes that one can iterate over and add to.&rdquo;</p>

<p>I&rsquo;m reminded of the term &ldquo;Gygaxian Building Blocks&rdquo;. I think I first heard this phrase from <a class="account" href="https://rollenspiel.social/@settembrini" title="@settembrini@rollenspiel.social">@settembrini</a> in <a href="https://pesa-nexus.de/2020/06/03/episode-1-wir-spielen-add-dmg/">episode 1</a> of <a href="https://pesa-nexus.de/category/zock-bock-radio/">Zock Bock Radio</a>, a mostly German, sometimes English podcast about role-playing games. &ldquo;Gygaxian Building Blocks&rdquo; are the things one can add individually to the rules and therefore everybody can participate in the game as a creator by writing spells, items, monsters, classes, and so on. These things stand on their own and aren&rsquo;t usually influenced by other such blocks. Adding them is easy. Conversely, the core game without the lists of spells, items, monsters and classes can be relatively short. It&rsquo;s a beautiful way of organizing the game.</p>

<p>For a sandbox campaign, the things that have that same quality and which I&rsquo;d call my &ldquo;sandbox campaign building blocks&rdquo; would be:</p>

<ul>
<li>powerful people: names, titles and retainers including spells and magic items</li>
<li>settlements: names, a small number of interesting buildings such as defensive structures and temples, the number of people ready to fight in case of a battle</li>
<li>ruins: mostly dungeons, big or small, often just three to five zones that may or may not map to &ldquo;rooms&rdquo; with monsters and treasure</li>
<li>magic schools: these are spell collections by legendary magic users that facilitate writing up spell selections for other magic-users since I can select the school they&rsquo;re from and pick an appropriate subset of spells (this is what my <a href="Spellcasters">Spellcasters</a> project is all about)</li>
<li>secret societies: names, leaders, and then for each settlement figure out if the secret society has a presence and if so, who the representative is</li>
<li>gods &amp; demons and their temples: names, themes, and then for each settlement figure out if there is a shrine or temple and if so, who the head priest is</li>
<li>monsters: I usually like to focus on a subset that is intelligent, with settlements and plans of their own, almost always humanoids</li>
<li>legendary items: as a consequence of treasure allocation for the above, some powerful magic items entered the sandbox and I need them on a separate list so that there can be rumours and quests pointing players towards them</li>
<li>permanent portals to other planes: names of these realms, powerful beings that can be reached (for spells, prayers, summonings‚Ä¶), environmental effects, rumours and pointing players towards them, preparing for the high level game</li>
<li>rough events in the past: names of wars, sides, immortal necromancer kings &amp; queens, but only insofar as they help flesh out locations and magic items etc. ‚Äì there is no need for a timeline and years, as far as I am concerned (trying to imagine what I would know without schools and books ‚Äì practically nothing except for &ldquo;Germans bad because of the World War&rdquo; and that&rsquo;s it, I guess?)</li>
</ul>

<p>Anyway, I think those are the things I am concerned about when I start a new campaign. If you have a blog post with a similar topic, let me know and I&rsquo;ll add a link!</p>
