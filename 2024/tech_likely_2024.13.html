Another day, another paean of praise for the Amiga‚Äôs 1980s pre-emptive multitasking GUI

date: 2024-03-24, from: Liam on Linux

Yes, the Amiga offered a GUI with pre-emptive multitasking, as early as 1985 or so. And it was affordable: you didn‚Äôt even need a hard disk.

The thing is, that‚Äôs only part of the story.

There‚Äôs a generation of techies who are about 40 now who don‚Äôt remember this stuff well, and some of the older ones have forgotten with time but don‚Äôt realise. I had some greybeard angrily telling me that floppy drives were IDE recently. Senile idiot.

Anyway.

Preemptive multitasking is only part of the story. Lots of systems had it. Windows 2.0 could do preemptive multitasking ‚Äì but only of DOS apps, and only in the base 640kB of RAM, so it was pretty useless.

It sounds good but it‚Äôs not. Because the other key ingredient is memory protection. You need both, together, to have a compelling deal. Amiga and Windows 2.x/3.x only had the preemption part, they had no hardware memory management or protection to go with it. (Windows 3.x when running on a 386 and also when given >2MB RAM could do some, for DOS apps, but not much.)

Having multiple pre-emptive tasks is relatively easy if they are all in the same memory space, but it‚Äôs horribly horribly unstable.

Also see: microkernels. In size terms, AmigaOS was a microkernel, but a microkernel without memory protection is not such a big deal, because the hard part of a microkernel is the interprocess communication, and if they can just do that by reading and writing each other‚Äôs RAM it‚Äôs trivially easy but also trivially insecure and trivially unstable.

RISC OS had pre-emptive multitasking too‚Ä¶ but only of text-only command-line windows, and there were few CLI RISC OS apps so it was mostly useless. At least on 16-bit Windows there were lots of DOS apps so it was vaguely useful, if they‚Äôd fit into memory. Which only trivial ones would. Windows 3 came along very late in the DOS era, and by then, most DOS apps didn‚Äôt fit into memory on their own one at a time. I made good money optimising DOS memory around 1990-1992 because I was very good at it and without it most DOS apps didn‚Äôt fit into 500-550kB any more. So two of them in 640kB? Forget it.

Preemption is clever. It lets apps that weren‚Äôt designed to multitask do it.

But it‚Äôs also slow. Which is why RISC OS didn‚Äôt do it. Co-op is much quicker which is also why OSes like RISC OS and 16-bit Windows chose it for their GUI apps: because GUI apps strained the resources of late-1980s/very-early-1990s computers. So you had 2 choices:
