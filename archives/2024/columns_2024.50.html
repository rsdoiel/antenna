ed is the standard editor, they say. I have strange fascination with it.

ed is a window in the past. This is a line-based editor, not a screen-based editor. It can be used with a line-printer instead of a screen. This also means that working with it is very different. I‚Äôll try to collect some interesting command sequences on this page.

I use ed to send mail from the command line. One way I do this is by using mail on the server. It‚Äôs a line-oriented mail client. You can use any editor with it, but the default feels very close to ed.

As you can see, it‚Äôs growing on me. üòÖ

Documentation

@mwl. wrote Ed Mastery which motivated me to get into ed.

Create a new file

Start ed without a file name. You‚Äôll be editing ‚Äúlines‚Äù in a ‚Äúbuffer‚Äù.

The commands to use:

  • i to insert a few lines
  • . on a line by itself ends the insertion of lines

alex@melanobombus ~> ed
i

This is a letter from me to you.
.
w hello.txt
47
q

Edit an existing file

When we provide ed with a filename, it is read into the buffer (and the number of bytes read are printed). The current line is the last line of the buffer.

  • a appends new lines after the current line
  • . on a line by itself ends the appending of lines

Now, ed is line-oriented and so we can provide line numbers and line ranges to our commands. If you omit a line number or the range, these commands apply to the current line only. Thus, a single comma means ‚Äúall the lines‚Äù. $ is always the last line.

  • p prints the lines
