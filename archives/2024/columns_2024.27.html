2024-06-30 Connecting the laptop to the server using UUCP

I recently connected my server to another server via UUCP. Now I‚Äôm thinking that perhaps I‚Äôd like to connect my laptop as another node to it. I probably don‚Äôt need it, because I connected the laptop to the server via NNCP. But still. Perhaps @lkh still wants to do stuff via UUCP. Perhaps it is the completionist in me.

For reference, see UUCP over SSH by @bortzmeyer. Since we‚Äôre using SSH, there will be a key pair, and since we‚Äôre using UUCP, we‚Äôll still a username and password. It‚Äôs going to be both.

I know, why aren‚Äôt we using NNCP.

Generate the key pair. 

cd /var/spool/uucp
sudo mkdir .ssh
sudo chown uucp:uucp .ssh
cd .ssh
sudo -u uucp ssh-keygen -C "uucp@melanobombus" -f id_rsa

The file uucp-key.pub now has the public key.

On the server (!), add a line to /var/spool/uucp/.ssh/authorized_keys, replacing the public key: 

restrict,command="/usr/sbin/uucico -u Umelanobombus -l" ssh-rsa KEY_FROM_ID_RSA_PUB uucp@dwalin

We will be using the username Umelanobombus to connect to the server. This is not a real user!

On the server, save username and password in /etc/uucp/passwd. 

Umelanobombus *secret*

On the laptop, save system name, username and the same password in /etc/uucp/call. 

sibirocobombus Umelanobombus *secret*

Now we have both levels: the public and private keys that allow the uucp user to connect via ssh ‚Äì and the username and password that is used by uucp.

Define our own name in /etc/uucp/config on the laptop. 

nodename melanobombus

Let‚Äôs add the system to /etc/uucp/sys. That is, on the laptop, define the server we‚Äôre going to call. Since I already know that sibirocobombus will report itself as campaignwiki, I‚Äôm setting up an alias. The commands line is important because by default only rmail and rnews are allowed but I want to allow rsmtp, too. The chat script waits for the ‚ÄúPassword:‚Äù prompt and then sends it. Note that this chat script is much shorter than the one used in ‚ÄúUUCP over SSH‚Äù, linked above. Apparently there‚Äôs no ‚ÄúLogin:‚Äù prompt on my server so I had to change it. 

system sibirocobombus
alias campaignwiki
commands rmail rnews rsmtp
call-login *
call-password *
time any
chat Password: \P
chat-timeout 30
protocol i
port ssh_campaignwiki

Add the new port to etc/uucp/port. Note that I‚Äôm listening to incoming ssh connections on a non-standard port so I have to provide. Also note that here we‚Äôre defining the uucp ‚Äúport‚Äù with a non-standard ssh ‚Äúport‚Äù. Just in case you‚Äôre not already confused. 

port ssh_campaignwiki
type pipe
command /usr/bin/ssh -a -x -q -p 882 uucp@campaignwiki.org
reliable true
protocol etyig

Before doing our first test, make sure that the server is a ‚Äúknown host‚Äù on the laptop, for the uucp user. Note how I am not using -q because I want to see the prompt!

The password to give on the prompt is the one in /etc/uucp/call.

Model Extraction from Neural Networks

date: 2024-07-01, updated: 2024-06-28, from: Bruce Schneier blog

A new paper, ‚ÄúPolynomial Time Cryptanalytic Extraction of Neural Network Models,‚Äù by Adi Shamir and others, uses ideas from differential cryptanalysis to extract the weights inside a neural network using specific queries and their results. This is much more theoretical than practical, but it‚Äôs a really interesting result.

Abstract:

Billions of dollars and countless GPU hours are currently spent on training Deep Neural Networks (DNNs) for a variety of tasks. Thus, it is essential to determine the difficulty of extracting all the parameters of such neural networks when given access to their black-box implementations. Many versions of this problem have been studied over the last 30 years, and the best current attack on ReLU-based deep neural networks was presented at Crypto‚Äô20 by Carlini, Jagielski, and Mironov. It resembles a differential chosen plaintext attack on a cryptosystem, which has a secret key embedded in its black-box implementation and requires a polynomial number of queries but an exponential amount of time (as a function of the number of neurons). In this paper, we improve this attack by developing several new techniques that enable us to extract with arbitrarily high precision all the real-valued parameters of a ReLU-based DNN using a polynomial number of queries and a polynomial amount of time. We demonstrate its practical efficiency by applying it to a full-sized neural network for classifying the CIFAR10 dataset, which has 3072 inputs, 8 hidden layers with 256 neurons each, and about 1.2 million neuronal parameters. An attack following the approach by Carlini et al.¬†requires an exhaustive search over 2^256 possibilities. Our attack replaces this with our new techniques, which require only 30 minutes on a 256-core computer‚Ä¶
