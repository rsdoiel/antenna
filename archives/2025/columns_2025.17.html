2025-05-03. I wrote a little Perl script called scr that opens a bunch of files and allows me to navigate using p for paragraph, s for sentence and w for word. Typing the letter with an optional range:

  • p prints the current paragraph, one sentence per line
  • s prints the current sentence
  • w prints the current word

Use an address prefix like ed:

  • a number prints the element with that number (currently zero indexed because I was lazy)
  • $ prints the last element (the first one is 0 but the last one requires counting)
  • plus and a number or minus and a number prints the element relative the current one (so +1p prints the next paragraph)

Use address prefixes like ed:

  • two such numbers separated by a comma prints the range (so 0,2s prints the first three sentences)
  • if you use a comma but just one number the other number is replaced by the current address (so ,+2w prints the current word and the next two)
  • if you use just the comma, this prints all the elements (so ,p is the same as 0,$p and prints the whole file)

Currently whitespace is not significant:

  • multiple whitespace characters collapse
  • leading and trailing whitespace disappears
